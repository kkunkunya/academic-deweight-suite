# 多 agent 辩论 Prompt 骨架

> 2026-04-23 新增。上游认知：`[[多agent辩论诊断法]]`。
> 用途：顽固 ≥50% 片段或章节级母骨架嫌疑时，启动三视角并行诊断。
> 三视角必须独立工作，禁止彼此复述，输出必须可落地到段号。

## 启用时机

| 条件 | 动作 |
|---|---|
| 初诊且无章节级骨架嫌疑 | 走主代理三侧面自审即可，不启用本 prompt 骨架 |
| ≥3 轮改写后 AIGC 仍 ≥50% | 强制启用三视角并行 |
| 章节目录出现 parade / 等量对偶 / standalone takeaway 密集 | 强制启用三视角并行 |
| 稿件长度 >5000 词且怀疑多分布 hybrid | 建议启用三视角并行 |

## 三视角 Prompt 模板（共享结构）

三视角共享以下 5 段结构，只替换 `[视角定义]`：

```
[视角定义]（三选一，见下方三段）

[上下文]
- 论文主题 + 作者身份 + 学科：
- 之前 N 轮降重的 AIGC 轨迹：
- 当前卡在哪一章 / 哪一节，百分比多少：
- 已尝试过的手术动作：

[被诊断的段落全文]
- 段号从 [X] 开始，每段前标段号
- 引用 [1]-[n] 必须保留
- 斜体 / 粗体原貌保留

[输出要求]（3-5 个具体任务，段号级，禁用模糊话）
1. ...
2. ...
3. ...

[字数约束 + 格式]
- 1200-1500 字，markdown
- 每个诊断必须锁到段号
- 不要客套、不要给改写全段，只给诊断
```

## 视角 1：检测器红队（`reviewer` subagent）

`[视角定义]`：

> 你是一个 AIGC 检测器的**红队**（Red Team）。不要降重，扮演知网/维普/GPTZero 等检测器的逻辑来**抓 AI 指纹**。

`[输出要求]`：

1. **Top N 最像 AI 的"露馅句"**（N ≤ 10）：给段号 + 原句节选 + 检测器会标记的特征（burstiness 低 / perplexity 均匀 / 模板节奏 / 教科书式推论链 / cleft 句型 / 破折号节奏 / 格言式开场 / 四元并列形容词等）+ 置信度（高/中/低）
2. **章节级分布诊断**：锁定每章"最严重的那种"AI 指纹（贴源综述骨架？高概率评价短语？破折号节奏？元话语密度？standalone takeaway？）
3. **机制级解释**：对关键发现给出"检测器的哪条通道会亮灯"（stance-engagement 联合通道 / n-gram 对称性 / 风格一致性 / humanizer 残片等），禁止"看起来像 AI"的模糊话
4. **反诊断**：如果之前轮次有"本来想救的动作"（插入第一人称 / 加破折号 / 切短段），判断这些动作反而强化了哪种指纹

## 视角 2：真人学术写作者（`doc-worker` subagent）

`[视角定义]`：

> 你扮演一个真人学术写作者——正常水平、带几个月阅读、不靠 AI 起草。具体身份按稿件类型匹配（英专本科生 / 工科硕士 / XX 领域 postdoc）。你的任务不是降重，是当诊断镜子：对比"一个真实 XX 身份的作者会怎么写这段"和"现在稿子的样子"。

`[输出要求]`：

1. **过度自洽的代价**：一个正常 XX 身份作者，会不会像当前稿子每一段都做 Z 动作（自我定位 / 边界声明 / 元反思）？还是说他更常见地做什么笨拙动作（信息倾倒 / 评价不均 / 忘记回指材料）？
2. **真人几乎不会写的"编辑腔"句**：给 5-8 个例子，每个说清楚"真人会怎么说"（直接给替代句或说"直接删"）
3. **健康的不洁痕迹清单**：真实本科/研究生论文里常见但这份稿子里全没有的东西（术语松用 / 文献跳跃 / 同一作者重复回头引 / 1.5 句话的段落 / 一段塞三论点 / 忘了回扣材料 / 不回应显而易见的反驳）
4. **元反思第一人称的假人味**：指出现在稿子里的 "I / my / we" 哪些是 **authorial**（保留）、哪些是 **editorial**（删除或改客观）

**禁用词**：降重、AIGC、AI 检测——你就是一个读过很多论文的人，判断"读起来像真人写的吗"。

## 视角 3：骨架审稿（`reviewer` subagent）

`[视角定义]`：

> 你是降重 skill 的**骨架侧审稿人**。不看句子好不好，看**段落之间的骨架、段内的节奏骨架、章节间的结构骨架**是否还暴露 AI 模板分布。

`[输出要求]`：

1. **按 A 编号命中诊断**：每个诊断必须锁到 A7/A12/A15/A17/A18/A20/A21/A30/A33/A35/A37 中的具体编号 + 具体段号证据，不允许抽象概括
2. **复合命中辨析**：某段可能同时命中多条 A，说明"主命中"和"次命中"的分工
3. **母骨架判定（关键）**：==如果只能打碎一个骨架就能让这几章同时下降 10+%，你会打哪个==？给出：
   - 骨架名称（A 编号）
   - 为什么它是"母骨架"（其他 AI 信号都是这个母骨架的表现）
   - 打碎的最小动作（具体到删哪几段 / 合并哪几段 / 哪几段改成不同形态）
   - 必须给一个明确答案，不要"可能是 X 也可能是 Y"

## 整合规则（主代理）

三视角产物汇总后主代理做决策：

1. **共识即根因**：三视角同时指向同一段号或同一骨架 → 直接写入主修方案
2. **冲突按证据侧优先**：证据侧（Gate 0 不变量 + A35 贴源）> 综合取舍层 > 其他侧面
3. **独有发现降级观察**：只有单视角看到的模式 → 不立即落手术，进入下轮观察 backlog
4. **集体盲区警觉**：三视角共享同一方法论预设（如"A30 是普适母骨架"）时可能集体失准——如果母骨架手术在某章节反效，触发 `[[章节级降重策略差异化]]` 的二次诊断

## Claude 侧执行入参示例

Claude 主代理并发 3 次 Agent 调用：

```
Agent({
  description: "检测器红队视角审 ChX/ChY",
  subagent_type: "reviewer",
  prompt: "[视角 1 模板填入具体上下文和段落]",
  run_in_background: true
})

Agent({
  description: "真人学术写作者视角审 ChX/ChY",
  subagent_type: "doc-worker",
  prompt: "[视角 2 模板]",
  run_in_background: true
})

Agent({
  description: "骨架审稿视角审 ChX/ChY",
  subagent_type: "reviewer",
  prompt: "[视角 3 模板]",
  run_in_background: true
})
```

三 agent 同时 in background 发起，等三份返回后再整合。

## Peer 侧执行入参示例

优先走 `knowledge/agent-mailbox/{claude-code,codex}/inbox/` 文件信箱协议；旧项目可兼容 `knowledge/codex-inbox/`。单份 inbox 任务让接收方内部并发 3 个诊断 subagent：

```markdown
# Peer 任务：对 ChX/ChY 做多 agent 辩论诊断

## 背景
[同上下文段]

## 被诊断的段落全文
[段号 + 段落内容]

## 你要做的事
在你内部 spawn 三个并行诊断 subagent，每个 subagent 用下面一种视角独立工作：

### 视角 1：检测器红队
[视角 1 模板完整段，包括输出要求和字数约束]

### 视角 2：真人学术写作者
[视角 2 模板]

### 视角 3：骨架审稿
[视角 3 模板]

## 整合要求
三视角各自产物写完后，由接收方主控 agent 做整合：
1. 共识条目（三视角都指向的）
2. 证据侧优先裁决的冲突条目
3. 单视角独有观察（进 backlog）
4. 母骨架判定（从视角 3 骨架审稿采纳）

## 返程契约
把三视角原始产物 + 整合结论写到对应 outbox：
- 新结构：`knowledge/agent-mailbox/<peer>/outbox/<same-filename>.md`
- 兼容结构：`knowledge/codex-outbox/<same-filename>.md`

内容包含：
- 三视角各自的完整报告（按视角 1/2/3 分段）
- 整合结论（共识 / 冲突裁决 / backlog / 母骨架）
- 对 Active Lead 下一步手术的建议

写完后回一句"outbox 已写"。
```

## 拓扑选择（详见 `multi-agent-topologies.md`）

- **默认**：在当前主代理内 spawn 3 视角（Claude 侧或 Codex 侧都可）
- **高价值片段**：跨模型对称辩论（Claude 三视角 + Codex 三视角互审）
- **成本控制**：主诊 + 外审（Claude 三视角 + Codex 1 个外审）
- **模型分工**：不同视角分派给不同模型（如红队走 GPT、真人视角走 Claude、骨架走 Gemini）
