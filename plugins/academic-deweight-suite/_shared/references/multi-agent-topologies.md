# 多 agent 辩论执行拓扑

> 2026-04-23 新增。上游认知：`[[多agent辩论诊断法]]`。
> 四种拓扑按"外部视角纯度"和"执行成本"做取舍，选一种即可，不叠加第二种。

## 四种拓扑对照

| 拓扑 | 结构 | 外部视角纯度 | 成本倍数（相对单视角） | 适用场景 |
|---|---|---|---|---|
| **A 单模型内三视角** | Claude 三 subagent / Codex 三 subagent | 中（同模型权重） | 3x | 日常诊断、中等难度顽固片段 |
| **B 跨模型对称辩论** | Claude 三视角 + Codex 三视角，两组产物互审 | 高（双模型权重） | 6-7x | 关键轮次、高价值片段（学位论文定稿前、投稿前） |
| **C 主诊 + 外审** | Claude 三视角主诊 + Codex 1 个 agent 做外部盲审 | 中高 | 4x | 想控制成本但要独立把关 |
| **D 模型分工** | 红队 → GPT / 真人视角 → Claude / 骨架 → Gemini | 高（三模型权重） | 3x（但需多模型接入） | 有多模型 Agent handoff 能力的场景 |

## 选择决策流程

```
1. 这批顽固片段的价值有多高？
   - 学位论文定稿 / 投稿前 → 走 B 或 D
   - 中间轮次迭代 → 走 A 或 C
   
2. 有没有跨模型 handoff 能力？
   - 只有 Claude 或 只有 Codex → 走 A
   - 两个都有，但 Codex 信箱延迟可接受 → 走 B 或 C
   - 有多模型 MCP 或 Agent handoff → 可以走 D
   
3. 对上一轮的结论有多少怀疑？
   - 不太怀疑，只是补诊 → 走 A
   - 怀疑上一轮集体盲区 → 走 B（双模型可相互指出盲区）
```

## 拓扑 A：单模型内三视角（默认）

Claude 侧：主代理并发 `Agent()` 三次，拿到三份报告后整合。
Codex 侧：一份 inbox 任务，Codex 内部 spawn 三 subagent，整合后写 outbox。

**优点**：简单、成本可控、无跨模型通信开销。
**风险**：三视角共享同模型权重，仍可能集体失准（如private local case disagreement）。

## 拓扑 B：跨模型对称辩论

```
1. Claude 执行拓扑 A（三视角并行 → Claude 整合）
2. Codex 执行拓扑 A（三视角并行 → Codex 整合）
3. Claude 把两份整合结论放在一起，做最终裁决：
   - 共识条目直接采纳
   - 冲突条目按 Gate 0 证据层优先
   - 分歧归因（是术语差异、视角差异，还是模型权重差异）
```

**优点**：真正独立的双模型视角，能互相指出对方的盲区。
**风险**：成本 6-7x；需注意两模型可能各自在同一方向上过度调整。
**触发**：学位论文定稿前、SCI/SSCI 投稿前、关键答辩材料定稿前。

## 拓扑 C：主诊 + 外审

```
1. Claude 执行拓扑 A（主诊三视角并行 → Claude 整合）
2. Claude 写一份 Codex inbox："这是我的主诊结论，你作为外部审稿人找问题。禁止重复我已发现的，只指出我漏看的 / 误判的 / 过度自信的"
3. Codex 1 个 agent 做外审，写 outbox
4. Claude 根据外审意见更新主诊结论
```

**优点**：外审是"补盲"而不是"复审"，信息增量最大。
**风险**：外审 agent 可能过度挑刺（为了显得有产出），需在 prompt 里明确"宁缺毋滥"。
**触发**：中等重要度的顽固片段、想要独立把关但不想跑双份完整诊断。

## 拓扑 D：模型分工

```
视角 1 检测器红队 → 走 GPT（检测器训练数据里 GPT 占比高，扮演检测器最本色）
视角 2 真人视角 → 走 Claude（Claude 对"真实人类写作质地"的感知更精细）
视角 3 骨架审稿 → 走 Gemini（长上下文 + 结构化分析能力）
```

**优点**：每个视角用最擅长该任务的模型权重。
**风险**：需要多模型 MCP / Agent handoff 架构；三份报告风格差异大需要主代理做风格统一。
**触发**：已有稳定的多模型编排基础设施（如 LangGraph / OpenRouter Agent 能力）。

## 通用护栏（所有拓扑）

1. **禁止三视角彼此复述**：每个视角的 prompt 必须包含"不要给出其他视角范畴的结论"（红队不做真人视角诊断、真人视角不抓检测器通道等）
2. **禁止给改写全段**：三视角只做诊断，改写是主代理后续步骤
3. **必须锁段号**：所有诊断必须能追溯到具体段号/原句，不允许抽象总结
4. **整合层由主代理做**：三视角不互相协商（保持独立性），整合由 spawn 它们的主代理做
5. **集体盲区警觉**：若主代理发现三视角在某段落上过度一致（都用同样术语、同样机制推论）— 这可能是方法论预设的集体投影，需启动 `[[章节级降重策略差异化]]` 的二次诊断
