# 模式目录（V4 / 2026-04-23）

在诊断与改写阶段使用本目录。只处理源文本中真实命中的模式。

## 元判断：战场迁移（2026-04-23 新增）

2025-2026 检测与合规目标从"原生 AI 文本"转向"被处理过的 AI / 人机混写 / policy-violating text"。改写目标从"压低 AI 味"扩展为：压低 AI 味 + 不留改写接缝 + 不露分布逆变 + 过 hybrid 检测 + 满足 provenance 披露。本文件顶部的前置步骤（执行优先级段的 0a/0b）和 A34-A37 都围绕这个迁移展开。

## 0. 底层去感原则

- 去 AI 感的重点不是换同义词，而是压低高概率模板分布。
- 优先处理四类高显著信号：去平均化、去支架化、增局部性、增非对称性。
- 能改成直接命题就不要保留抽象宏词；能落到局部对象就不要停在全景总括。
- 局部具体性只能来自源文本已给出的对象、条件、方法、结果或边界；无支撑时删空话，不补细节。
- 改写方向参见 `references/human-writing-anchors.md`：目标是让表达分布更接近真实学术作者，不是让文本"看起来像被好编辑改过"。

### 2026-04-23 新增元观察（status: signal）

以下 3 条不是独立模式（人工难以单独诊断），作为所有 A 族改写时的元警觉：

- **分布逆变警觉**：某段看起来像"从另一种表达分布翻回来的"本身就是信号。逆变后句式"过分通顺"反而可疑——真实改写应保留一些表达颗粒度差异。避免把所有 AI 短句都打磨到学术体，允许某处保留略粗糙但可读的表达。
- **多样性波动**：真实学术写作的 surprisal 和词汇丰富度在段内段间都有自然波动。波动塌缩（各段分布过平）本身是 AI 信号，不是"风格统一"的优点。改写时允许部分段落显得用词不那么精致但信息够。
- **依存骨架差异**：句法结构（主从句嵌套深度、依存链长度）应有自然差异。连续多段依存骨架过规整是 AI 信号——即使措辞不同骨架一致仍高概率。改写时注意让句法骨架有波动。

## A. 强去痕模式（Aggressive De-AI）

### A1 空泛意义夸大
- 检测：`标志着/彰显/pivotal/testament/landmark`。
- 改写：改为直接贡献、方法或可验证结果。
- 禁止：杜撰指标。

### A2 宣传式形容词
- 检测：`groundbreaking/revolutionary/cutting-edge/卓越/前所未有`。
- 改写：删除营销词，仅保留可核验技术陈述。

### A3 模糊归因
- 检测：`studies show/有研究表明/experts believe`。
- 改写：无引用时保守中性表达；不得编造作者与年份。

### A4 公式化挑战-前景结尾
- 检测：`despite challenges... bright future/尽管...前景光明`。
- 改写：改为具体限制 + 直接下一步。

### A5 填充短语
- 检测：`值得注意的是/in order to/at this point in time/it should be pointed out`。
- 改写：删除填充，只留命题。

### A6 过度 hedging
- 检测：`could potentially possibly/might perhaps`。
- 改写：压缩为单一校准 hedge（`may`、`可能`）。

### A7 机械并列（三项对称 + 散文化目录列举）
- 检测：① 反复三项并列、句骨架重复；**尤其注意隐式三段平行**：即使去掉"其一/其二/其三"等显式标记词，若三段仍具有对称的切入角度、类似的句式和等量信息，仍属于 A7 命中。替换标记词（如改为"编码层面…概率层面…种群层面"）不等于打散结构。② **散文化目录列举（≥4项）**：散文中连续出现≥4项系统组件/模块/章节，每项恰一句且格式高度等量，即使无编号也属于 A7 命中。
- 改写：在语义不变前提下打散节奏；允许非对称句式；不同段落允许详略差异与切入角度差异。**散文化目录列举的处理**：只详写1-2个关键项，其余合并为"其余模块见图X"或移入表格；破坏"每项一句"的等量结构——某项用两句加机制说明，某项只用半句从句带过。
- **修复策略（叙事流化）**：去掉每段的"X层面/X角度/X维度"标签，改为由前段内容自然推进到后段；允许某段无显式主题句、某段仅作过渡、某段结尾不收归因果——用不完整感打破"每段=完整论证单元"的结构均匀性。

### A8 系词回避
- 检测：习惯性 `serves as/functions as/acts as`。
- 改写：可改为更直接的 `is/are`（仅在语义等价时）。

### A9 Chatbot 痕迹
- 检测：`Great question`、`I hope this helps`、`当然可以`、`以下是` 等。
- 改写：全部清除。

### A10 视觉风格噪音
- 检测：emoji、过度加粗、破折号滥用、装饰性列表头。
- 改写：统一为朴素学术格式。

### A11 高频 AI 连接词堆叠
- 检测：`此外/同时/进一步/Furthermore/Moreover` 高频串联。
- 改写：合并或删除冗余连接词，保留必要逻辑关系。

### A12 句长同构
- 检测：连续 3 句以上长度与结构高度相似。
- 改写：执行长短句交错与语法重排，不改事实。

### A13 刻意换词（同义词循环）
- 检测：同一对象被反复换称，如 `该方法/该框架/该方案/该机制` 交替出现但指向同一实体。
- 改写：能重复就重复，以最清晰、最稳定的术语为准。

### A14 虚假范围与全景总括
- 检测：`从 X 到 Y`、`覆盖整个格局/全景/广泛层面`，但未落到有意义的学术尺度。
- 改写：压回具体对象、阶段、任务或问题域；删去无信息增量的范围铺陈。

### A15 论证结构模板化
- 检测：段落结构反复呈现总-分-总或总-分模式；每段都有"首句总括-中间展开-末句小结"完整结构。
- 改写：按论证需要保留必要结构，打散无功能的段落框架；允许只有一句话的段落或无显式主题句的段落。

### A16 语气均匀性
- 检测：全文确信度高度一致，无论命题强度是否相同；hedge 词与断言词分布无差异。
- 改写：对证据强的命题用更直接语气，对不确定命题保留局部犹豫；不要让全文的认识论信号均匀平滑。

### A17 面面俱到的覆盖感
- 检测：各子话题被赋予近似等量篇幅，或每个相关维度都给出评价，读起来像系统性目录。
- 改写：依作者判断取舍篇幅；允许主要重点详写、次要话题一句带过或不写；去掉篇幅平衡带来的人工感。

### A18 显式逻辑黏合剂过密
- 检测：`此外/同时/进一步/Furthermore/Moreover/In addition/Additionally` 每 2-3 句出现一次，或密度明显超过中等学术文本的自然水平。
- 改写：删除多余连接词，让相邻句子通过内容逻辑直接衔接；只在真正需要显式标记转折或推进时保留。

### A19 抽象层次平推
- 检测：连续 3 句以上停留在同一抽象层（均为高度概括，或均为具体描述），缺乏层次切换。
- 改写：引入抽象-具体之间的跳跃；具体对象与高层命题交替出现，使文本有深度感而非平铺感。

### A20 操作-评价耦合（句对级 + 段落级）
- **句对级检测**：描述一个操作或参数设置后，立即紧跟一句肯定性价值判断（如"取得平衡""可有效防止""确保…不因…丢失"）。
- **段落级检测**：整段论证目的为"描述做了什么 → 所以达到了什么效果/有何意义"，即使两者不相邻也属于命中；整段读起来是"操作+立即正面评价"的因果直线。
- **改写（句对级）**：删除紧跟的价值判断句，或将价值判断移至段尾统一说明一次；操作描述本身保持不变。
- **改写（段落级）**：将评价/意义部分移到更远的位置（如下一段开头或章节末尾统一讨论），或在操作与评价之间插入限制条件/反面边界，打破"操作→直接正面评价"的因果直线；允许某些段落只有操作描述而无显式评价。

### A21 列表条目格式同构
- 检测：假设条件、创新点、约束条件等编号列表中，每条都遵循完全相同的格式（如 `**关键词**：一句说明；一句后缀`），无任何详略差异，读起来像从模板批量生成。
- 改写：人为制造格式差异——某几条省略后缀解释、某几条多加一句工程理由、某几条只用一个短句；让条目详略服从内容重要性，而非格式整齐性。
- 注意：当内容本身强制要求一致格式（如参数符号表、约束方程组）时，A21 不适用。

### A22 二元数字精确镜像
- 检测：两元素构成完整对偶，且括号内数字互补（如 `后30%，60个` vs `前70%，140个`）；两侧描述骨架相同，末尾价值判断也对称。
- 改写：①将例子或额外说明嵌入其中一侧，另一侧描述缩短；②去掉两侧括号内的精确数字，或只保留一侧的量化注释；③两侧结尾改为不同类型（一侧用机制说明，另一侧用目的说明），打破收尾对仗。

### A23 粗体并列标题组
- 检测：多个（≥3个）具有相同格式的粗体条目标题，每条后跟近似等长内容，形成高度视觉对称的多段区块。
- 改写：①合并可合并的项；②保留有公式/数值的项单独成段，但去掉其粗体标题；③其余标题一律去掉，仅保留正文；制造"有的有标题、有的无标题"的非对称视觉感。

## A-扩展：结构与分布级模式

### A24 公式-散文耦合模板
- 检测：公式前后的解释文本遵循固定五段式：`引出句 → 公式 → "$X$ 为…；$Y$ 为…；$Z$ 为…"变量定义块 → 物理意义一句话 → 衔接下一节`；变量定义块的分号分隔列表在全文重复出现≥5次，节奏完全同构。
- 与现有模式的区别：A21仅针对编号列表；A12检测句长；A24是公式散文框架的段落级模板。
- 改写策略：①变量定义块中某些已在上文出现的变量省略不再解释，某些变量用括号嵌入公式行内；②打破五段式顺序：某处省略引出句直接给公式，某处把物理意义前置；③辅助公式只给结论，不展开变量定义；④允许不同公式段详略差异。
- 不可抗例外：变量符号表、参数列表属于A24的格式约束场景，不强制改写，降低优先级。

### A25 短密信息弹
- 检测：≤400字符的段落内压缩≥3个独立事实/数字/百分比，且每句信息密度近似相等（无主次、无节奏起伏、无过渡句）；全段无废话也无呼吸空间，"每句等权信息密度"是核心检测信号。
- 改写策略：①在中间插入一句机制解释/限制条件/过渡句，打破匀速节奏；②允许某些数字只在表格中出现，不在正文重复；③如果是摘要，减少百分比列举，改为"关键数字+定性描述"；④最核心的一个结论展开为两句，其余压为从句。

### A26 散文化目录列举
- 检测：散文中连续出现≥4项系统组件/章节/参数组，每项恰一句且句式高度等量（无编号，但结构同构），形如"[名称][动词][功能描述]"的隐式目录。
- 改写策略：①只详写1-2个关键项，其余合并（"其余模块通过图X接口连接"）；②将列举移入表格，正文只叙述模块间数据流；③破坏"每项一句"的等量结构。

### A27 数值密集型结果陈述
- 检测：一段内≥3个"指标名称+数值，较baseline改善X%"格式，且每个指标格式相同、节奏均匀。
- 改写策略：①核心指标在正文叙述，次要指标只引用表格；②打破"较X改善Y%"的重复句架：某处用绝对差值，某处用定性描述；③不在一句话内并列超过2个百分比改善值。
- 注意：不删除数字，只改叙述结构；Gate 0硬约束数字不可变。

### A28 结论/展望节级模板
- 检测：全文总结节按逐章回顾结构；未来展望节按"局限1→方向1，局限2→方向2，局限3→方向3"对偶循环，每条等长。
- 改写策略：①总结改为按主题线收束，不逐章复述；②合并同类局限，打破一一对应；③某个局限不给方向（承认无解），某个方向不基于已列局限；④允许展望详略不均。

### A29 中文本土对偶承接结构（status: signal / 2026-04-23 / 第三轮补充高频动词块）
- **核心诊断**：中文 LLM（Qwen3/DeepSeek/豆包/文心）用显性成对或多段关联词制造伪平衡结构，或用固定动词块填充空洞论断，区别于英文翻译腔。
- **检测**：
  - 两段对偶：`一方面…另一方面`、`既…又…`、`不仅…而且`、`在 X 方面…在 Y 方面`、`从 X 角度…从 Y 角度`
  - 多段承接：`首先…其次…最后`、`一是…二是…三是`、`其一…其二…其三`
  - **中文 AI 高频动词块**（2026-04-23 第三轮新增，待核验）：`体现了/反映了/提供了/彰显了/充分说明了/展现出/呈现出` + `一种/一个/某种` + 抽象名词。例如"体现了算法的优越性" / "提供了一种新的思路"。密度过高或每段都有 → 命中
  - 判定要点：即使删掉关联词，两/三段仍呈现对称的信息量、等长句式、平行句骨架。
- **与 A7 的分工**：A7 管"去掉标记词后的隐式对称"；A29 管"中文显性关联词 + 对称结构 + 高频动词块"的组合。
- **改写策略**：
  - ①**主线化**：用散文化推进替代对偶——某一侧作为主线展开，另一侧压进从句或只在必要时提一次。
  - ②**删关联词 + 让内容承载逻辑**：去掉"一方面/另一方面"，让两段内容本身形成自然转折或递进。
  - ③**打破信息等量**：某侧详写（给机制或边界），某侧仅一句带过；避免两段篇幅接近。
  - ④**允许不回收对偶**：开头抛出"两个方向"不等于结尾要各给一个方向；允许某一侧不闭合。
  - ⑤**高频动词块替换**：用具体动词替代空心搭配——`体现了 X 的优越性` → `在 Y 场景下比 baseline 快 Z 倍`；`提供了一种新思路` → `引入了 X 的处理机制`；`充分说明了有效性` → 直接删除或换为量化结论。或删空动词让主语承担动作。
- **典型触发场景**：摘要、引言对比、方法论比较、讨论节的优劣权衡。
- **不可抗例外**：并列方程组、对照实验的标准化描述（A/B 组、前/后对比）不强制改写。

## A-深层：顽固高分片段策略

> 适用场景：片段经≥2轮改写后 AIGC 仍≥0.85，表层模式（A1-A28）边际递减。

### A30 认知完整性均匀性消除（Perplexity均匀性 + 段落级认知闭合）
- **核心诊断**：全文 perplexity 均匀（每句都"完美成型"，无高低起伏）、认知密度一致（无论内容难易，每段的完整程度和清晰程度稳定不变）。更深一层：**每个段落都是一个自足的认知单元**——有明确目的、平滑逻辑流、干净结论——这本身就是最强的 AI 信号。真实人类学术写作有自然的认知杂乱：不完整的想法、插入的旁枝观察、句子之间的逻辑跳跃、作者中途改变论证方向的痕迹。
- **两层检测**：
  - 句级：每句话的清晰度、完整度高度一致，"每句都像精心修改过"。
  - 段级：每个段落都是"目的→展开→收尾"的完整小论文，无一例外。段落之间的独立性过高——删除任何一段不影响相邻段落理解。
- **改写策略**：
  - ①主动制造**认知速度差**：某处极快（一句话直接给结论，不解释），某处极慢（追加工程边界或判断）。
  - ②插入**作者判断语**（"这一设置并不直觉，但……"；"更严格的做法需要X，这里为简化采用Y"）——LLM自发生成中最稀缺的信号。
  - ③允许某段**刻意不完整**：提出问题不给答案，给出现象不给机制解释。
  - ④**段间依赖**：让某段的结尾成为下一段的起点（不是通过连接词衔接，而是通过论证内容的未完成感）。
  - ⑤**场景置入法**（针对≥4轮顽固片段）：彻底放弃"原因链/背景铺垫"叙事模式，直接从一个具体失败观察或反直觉现象切入，不用"当X时...导致Y"的条件句。
- **不可抗例外**：摘要、核心引理、Gate 0保护内容不适用。

### A31 教科书式概念复述陷阱
- **核心诊断**：一段文字的写作目的是"让读者理解X是什么"（解释型），而非"说明作者为何在此情境下选择X而非Y"（判断型）。LLM预训练语料中"正确解释X"的内容密度极高，任何清晰的概念解释都高度可预测。
- **检测特征**：段落包含完整的"定义→公式→参数解释→物理意义→与相邻概念关系"链，无作者立场；内容是领域基础知识。
- **改写策略**：
  - ①**内容重心转移**：将"解释X是什么"转变为"本文为何选X、X在本研究场景的具体边界与局限"。
  - ②**压缩标准解释**：已知的标准定义压缩至1-2句或移入表格，正文只保留"本文特有的配置判断"。
  - ③**加入工程判断**：不解释模型结构，改为阐述选型理由和场景边界。
  - ④**允许不展开**：对读者可自行查阅的标准内容，直接引文献——"SOC估计采用EKF，标准实现见文献，此处仅列接口参数"。
- **反升警告**：用"比较选型框架"替换教科书解释后仍可命中A31——"方案A有局限→方案B改进→方案C成本高→选方案B"本身是高可预测序列。根治：不从"有几种可选方案"开始，直接从场景约束切入，只保留本文采用方案的工程边界。

### A32 过度严密因果链（2026-04-23 第三轮从"消融分析"泛化）
- **核心诊断**：任何论证段出现"前提 A → 推理 B → 推理 C → 结论 D"式的完整无跳跃因果链，都属 A32 命中。**消融分析节是典型场景但不唯一**——推理模型（o3/QwQ/DeepSeek R1）的 Think-then-Answer 机制会在所有论证段都留下 Over-justification 痕迹。
- **原典型场景（消融实验）**："去除X → 指标Y恶化（具体数值）→ 机制M导致 → 因此X有效"。
- **泛化场景**（2026-04-23 第三轮新增 / status: signal / 来源待核验）：方法论推导段（假设→推导→中间步骤→结论）、讨论节解释段（现象→原因→机制→结论）、文献综述段（前人工作→局限→改进→本文贡献）——所有完整无跳跃的因果链。真实作者会省略"显而易见"的中间步骤（默会知识），呈现跳跃性。
- **改写策略**：
  - ①**反转叙事顺序**：从"观察到某个值得关注的行为模式"开始，而不是从前提起步。
  - ②**打断机制链**：只陈述观察到的行为，机制推断用弱语气。
  - ③**省略默会知识**：假设读者知道基础概念，不从头讲；中间推理步骤允许跳过一两步。
  - ④**异常视角**：不说"X有效是因为Y"，改为"去掉X后，策略在某个特定状态下表现出……这一现象在去掉Y时未出现"。
  - ⑤**禁止等量对比**：只详写贡献最大或最意外的一项，其余一句带过或只引表格。
  - ⑥允许某个结果/机制"原因暂不明确，待后续分析"。

### A33 段落间衔接过于顺滑（status: signal / 2026-04-23 新增）
- **核心诊断**：段与段之间的逻辑承接过于完美——主题平滑过渡、承接句密度均匀、相邻段的论证接缝无毛刺。与 A30 的分工：A30 说每段自己完美（段内），A33 说段与段之间衔接完美（段间）。溯源：知网 AIGC 3.0 新增"上下文关联分析"+"风格一致性检测"，衔接质地本身已成为检测信号。
- **两层检测**：
  - **承接句均匀**：每段开头都用"在此基础上/进一步地/基于上述/与此同时"等显式过渡词，密度稳定。
  - **主题无跳跃**：相邻段主题呈线性推进（A→A'→A''），从不出现突然切入、主题偏移、或"抛出问题→暂时搁置→下一段讨论别的→再下一段回来"的真实思维轨迹。
  - **接缝无毛刺**：每段结尾都优雅收束为下段铺垫；没有"结论突兀留白"、"未回答问题悬置"、"视角突然切换"这类真实作者的常见现象。
- **与 A30 的分工**：A30 检测段内自足（段内目的→展开→收尾完整）；A33 检测段间衔接（承接句均匀 + 主题线性 + 毛刺缺失）。改写时先 A30 处理段内，再 A33 处理段间——不要同步做（会造成过度改写）。
- **改写策略**：
  - ①**删承接句**：允许相邻段之间不做显式过渡，让读者自己推断关系。
  - ②**主题轻微偏移**：相邻段不严格沿一条线推进——某段临时拐到一个具体观察再回来，或结尾悬置一个问题到下下段才回应。
  - ③**结尾毛刺**：某段结尾不做收束性总结，直接抛出下段的前提；或结尾留一个没展开的观察。
  - ④**开头突兀**：某段不用"基于上述/进一步"等承接词开头，直接从一个具体对象或现象切入。
  - ⑤**允许视角切换**：从方法描述突然切到工程约束、从实验结果突然切到失败案例——真实论文的段落切换往往带有作者临时判断痕迹。
- **实测陷阱**：改写后可能出现"为求不顺滑而刻意生硬"——要以"保持论证推进"为底线，毛刺不是故意制造的破坏，是真实写作的自然痕迹。
- **不可抗例外**：摘要、定理证明链、方法流程图对应段落（有固有线性结构）降低优先级。

### A34 推理模式接缝（status: signal / 2026-04-23 新增）
- **核心诊断**：同一篇文本内混合出现不同推理深度的段落——某段明显"闭环"（显式穷举选项、排除备选、给出闭合推理链），某段明显"压平"（直接给结论不展开），两种模式并存但切换生硬。这是 reasoning-family 模型（GPT-5 router / Gemini 2.5 thinking / DeepSeek V3.1 Think-NonThink / R1 后处理）在同一次生成中留下的模式切换接缝。
- **与 A30/A31/A33 分工**：A30 管段内完美、A31 管教科书解释、A33 管段间衔接过滑；A34 管推理深度在段间/段内突变。
- **检测信号**：
  - 某段突然出现"一方面考虑 X，另一方面考虑 Y，综合来看选 Z"式的显式穷举+排除结构
  - 相邻段落"思考密度"明显不同——一段压缩、一段展开、一段又压缩
  - 某段出现"推理压平残片"：局部过度解释（本不需要解释处解释了）、突然性总结（节奏骤变）、语言层级跳变、异常收束
- **改写策略**：
  - ①**统一推理深度**：全文要么 polished（压缩叙述 + 引用展开），要么 deliberate（显式推理 + 选项比较），不交替使用。
  - ②**deliberate 段落聚团**：若必须保留 deliberate 推理段（方法论/算法节），集中在一组连续段落内，不要散布在 polished 叙述中。
  - ③**消除压平残片**：检查每段尾部是否有异常收束或突然性总结；改为平滑延续或自然断句。
  - ④**语言层级一致**：同一节内不混合学术体与口语化总结。
- **不可抗例外**：方法论/算法描述节本就可以保留 deliberate 推理；Abstract/Intro/Conclusion 应保持 polished。

### A35 生成式贴源改写（status: signal / 2026-04-23 新增）
- **核心诊断**：论证骨架、证据顺序、术语簇、因果走向仍贴着某篇 source（参考文献/综述/教材），但表面已换皮。典型场景：让 AI 基于某篇参考文献"改写/扩写一段文献综述"——表面看不出抄袭，但骨架几乎 1:1 对齐。
- **与 A31 分工**：A31 是"解释 X 是什么"（教科书式，来源是标准定义链）；A35 是"贴着某篇 source 重写"（来源是特定文献的论证骨架）。
- **检测信号**：
  - 段落论证顺序与某篇参考文献对应节完全同构
  - 证据（引用、数字、实验结果）出现顺序严格对齐某一 source
  - 因果连接词的位置与 source 节奏一致
  - 术语簇高度贴近某 source 的术语选择（而非领域通用分布）
- **改写策略**：
  - ①**重组论证骨架**：按作者自己的论证需要重排，不按 source 顺序。这是最有效的一步。
  - ②**证据交叉**：引用多篇 source，打破单篇 source 的顺序结构。
  - ③**打破因果节奏**：不在 source 的转折处转折。
  - ④**术语簇替换**：用领域通用术语替换 source 特有的术语组合。
- **反升警告**：单纯换词不破坏骨架，单纯换句式不破坏骨架；**只有重排论证骨架才真正去"贴源"感**。这和 A7（机械并列）打散的是句式不同，A35 改的是段内论证流。
- **不可抗例外**：方法复现段、baseline 对比段本就需要紧贴 source 描述；只对综述段/背景段/讨论段适用。

### A36 作者性抽薄（status: signal / 2026-04-23 新增 / 2026-04-23 补 editorial vs authorial 二分）
- **核心诊断**：作者作为论断主体从文本里退出——论断持续外包给"研究表明/数据显示/模型证明"，第一人称立场退回客观描述位，不做价值判断也不给选型理由。看起来客观，实际缺了一个会思考的人。
- **与 B1 的张力**（作者性对称边界）：
  - B1 管下限：不要过度个人化（不注入第一人称情绪、主观反应、采访细节）
  - A36 管上限：不要过度客观化（作者作为论断主体不能缺席）
- **Editorial stance vs Authorial stance 二分（2026-04-23 补，关键深化）**：
  - 插入第一人称不等于恢复作者性。第一人称分两种，**恢复前者、删除后者**：
    - **Authorial stance**（要补）：带命题 commitment 的 "I argue / we find / we adopt X because / we reject Y"。后续必须有 X 的辩护或反驳空间。真人作者的分布 = **unbalanced 尖峰式**，集中在方法论选型、关键判断、反驳回应处。
    - **Editorial stance**（要删）：对已有命题的元评论 "What my reading takes / which is why I am flagging / that is the shape of the opening / I carry this forward / I don't feel I'm forcing it / that is the map / sit honestly inside"。不增加命题，只在已有命题周围加航标。真人分布 = 每段 < 0.5 次，几乎不出现。
  - 2026 主流检测器的 **stance-engagement 联合通道**对这种 editorial stance 特别敏感：第一人称密度高但命题锚定数（claim-evidence 对）没同步升高、self-mention 和 hedging 高度共现——这是 LLM 被要求"写得像人类"时最稳定的 humanizer 残片指纹，**比完全没有第一人称还更像 AI**。
  - 检测操作：把全文所有 "I / my / we / our" 出现位置标出来，按类别分——
    - 有后续命题辩护或反驳空间的 → Authorial（保留或补齐）
    - 只是元评论、和 hedging 共现、分布均匀 → Editorial（删除或改客观陈述句）
  - 检测辅助句法信号：Editorial stance 高频出现在 what-cleft（"What my reading takes is..." / "What makes this case useful is..."）和 it-cleft 句里；真人 authorial stance 多在主语位（"I argue X" / "We adopt X"）。
  - **Private local case omitted from public package。
- **检测信号**：
  - 方法节全篇 passive voice，缺少"本文采用 A 的理由是..." / "we choose A over B because..."
  - 讨论节全篇"结果表明/实验证明"，缺少"我们认为/本文主张"
  - 参数选择、实验设计、阈值设定的理由不显示，只展示结果
  - 对比其他方法时只说"优于 baseline X%"，不说"我们预期这个方法更优的场景"
  - 反驳/限制/未解决问题段稀薄或缺失
- **改写策略**：
  - ①**关键节点插入作者立场**：方法选型、参数选择、结果解读的关键节点至少一次直接立场表达。
  - ②**部分"X 表明"改为"我们观察到 X"**：把论断主体从研究对象收回到作者。
  - ③**讨论节给出作者判断**：而不是纯 passive voice 的现象陈述。
  - ④**暴露限制与不确定**：主动写出"本文未验证的场景""这个结论在 X 条件下可能不成立"。
  - ⑤**不是注入情绪**：A36 恢复的是作者作为论断主体的在场，不是个人色彩。
- **不可抗例外**：Abstract 与纯 result 节可保持客观描述；Method/Discussion/Conclusion 不可过度客观——这三处是 A36 的核心触发场景。

### A37 混合作者接缝（status: signal / 2026-04-23 新增）
- **核心诊断**：一篇文本里存在两种或以上的写作分布（人写段 + AI 段 + AI polish 段），在分布接缝处可见突变。相邻段的句长、hedge 密度、术语精度、连接词密度、视觉风格明显不连续。
- **与 A33 的分工**：
  - A33 假设整篇同分布，检测段间衔接过于顺滑（太平滑 = AI）
  - A37 假设多分布混合，检测段间分布突变（突然不平滑 = hybrid）
  - 实际改写中先判断同分布还是混分布（见执行优先级 0a 步骤），再决定用哪个
- **检测信号**：
  - 某段 vs 相邻段的句长分布明显突变（一段短句密集，相邻段长句嵌套）
  - 某段 hedge 密度显著高/低于全文基线
  - 某段术语精度或领域术语密度明显不一致
  - 某段的认知完整性（A30）或衔接质地（A33）与相邻段不匹配——一段很 AI 味，相邻段很人味
  - 视觉可见：某段突然多粗体/emoji/装饰性列表，相邻段又回到朴素学术体
- **高影响句定位**（诊断操作）：
  - 取 5-10 个随机段落，对每段问"如果删掉最可疑的一句/一段，全文 AI 感是否明显下降？"
  - 多数答案"会"→ AI 感集中在少数高影响区域（典型 hybrid 或 humanizer 残片），改写从高影响区域优先
  - 多数答案"不会"→ 分布均匀（可能是真人，也可能是未洗原生 AI，结合其他维度判断）
- **改写策略**：
  - ①**统一全文分布**：定位最突兀的段落，按全文基线重写；通常目标是向"人写段"对齐。
  - ②**优先处理高影响句**：humanizer 洗过的文本典型是"整体中庸、局部露馅"——少数句子承担大部分 AI 感。逐段平铺改效率低。
  - ③**风格基线一致化**：节选全文最像作者自己风格的 2-3 段作为锚点，让其他段落向这几段靠拢（句长分布、hedge 风格、连接词习惯、术语选择）。
- **不可抗例外**：节间（Abstract/Intro/Method/Result/Discussion）本就有不同 register；A37 只抓**节内**分布突变，不抓节间正常差异。

### A38 精炼反思型学术风格 / punchy reflective academic（status: signal / 2026-04-23 第四轮新增）
- **核心诊断**：2024+ 主流 LLM（GPT-4 / Claude 3.5+）生成的"精修学术"风格——句子短促、常用冒号与破折号、喜欢 meta-commentary、喜欢 punchy takeaway 段落。旧模板（A28/A31）像百科全书，A38 像"被好编辑改过"——两者都是 AI 风格，检测器训练数据已把 A38 作为正样本，识别率甚至高于 A28。
- **溯源**：Private local case omitted from public package。
- **六类子模式**（本 A38 下位）：
  - **label + colon 卡片句**：`Heroic action: coordinated strikes ... on one side, unarmed rescue on the other.` 像 PPT speaker notes
  - **editorial beat 极短段**：`Evidence first. Framework second.` / `Five chapters.`——独立成段的两拍口号
  - **editor voice 元注释**：`That is the map.` / `What follows...` / `Start with X.` / `On the first question...`——编辑在组织页面的声音，不是研究者在解释判断
  - **tutorial staircase 教学楼梯**：`opener → score → expanded → mini takeaway` 同节内重复 3-4 次
  - **parade 排位巡游**：同一模板反复 4+ 次（`IDV does most / PDI second / MAS texture / UAI lightest`）
  - **aphoristic recap 金句收束**：单独成段的漂亮一句话（`IDV holds together.` / `Authority is not an obstacle here.`）
- **与 A28/A31/A36 的分工**：
  - A28 管老模板骨架；A31 管教科书式复述；A38 管新模板——结构上可能已无 A28 的对称骨架，句式层面仍是精修 AI
  - A36 说"作者退出"；A38 说"作者回来了但回来的是 editor 不是 researcher"
  - 清除 A28 后若 AI 率不降反升，查 A38
- **与 A37 的关系**：A37 识别段间分布突变（hybrid seam）；A38 是 A37 识别到的"AI 段"里的主流子类型，相当于 A37 的下位内容指纹
- **检测信号**：
  - 独立段 <18 词、无新证据、带冒号或破折号
  - 同节内连续 3+ 段使用同一 `label + colon` 或 `opener + score + takeaway` 模板
  - 通篇节奏"干净利落像被好编辑改过"，每段都有 topic sentence，句间衔接由冒号/破折号/短句承担
  - 存在独立成段的 aphoristic 金句（无新信息，只包装前文）
- **改写策略（反精修）**：
  - ①**删接缝**：<18 词+不承担新证据的独立段一律并入邻段或删除；label+colon 卡片不能独立成段
  - ②**反对称**：同一主题的多个子项不做等长展开——一个详写其余一句带过；parade 压成"一次详写 + 其余整合成 qualifier 段"
  - ③**标点节流**：每个正文段最多保留 1 个破折号或 1 个冒号，不与短句+分号叠加
  - ④**去 editor voice**：`That is X` / `What follows` / `Start with` / `On the N question` 句框全删；作者在场回到 decision point（为什么选 X / 为什么 Y 只做 qualifier），不做 page management
  - ⑤**留 unresolved edge**：不每次 `not X but Y` 漂亮闭环；limitations 段留一处"只说边界不立刻安抚"
  - ⑥**散文化收束**：aphoristic recap 塞回前段做尾句，不独立成段
- **不可抗例外**：
  - 真正的小节标题（Heading 3+）可以短
  - 结构性 enumerated list（RQ1/RQ2/RQ3）可以单行
  - Abstract 节的紧凑语体通常是正常 register

### A39 换风格陷阱 / LLM self-restyle trap（status: signal / 2026-04-23 第四轮新增 / 元规律）
- **核心诊断**：LLM 为降 AI 率做的风格切换，仍命中检测器——因为每次重写都用同一权重网络，换的是表层而不是分布本身。从一种 AI 风格（如 A28）移到另一种 AI 风格（如 A38），检测分数横移甚至反升。改写者会感觉"这轮更像自己要的样子"——而数字告诉相反故事。
- **机制**：
  - LLM 改写从**同一训练分布**采样——换风格只是在分布内部位移
  - 检测器训练数据持续扩充——2024-2026 的新 LLM 风格（A38 家族）已作为正样本进入主流检测器
  - 结果：LLM 的"风格选择空间"和"检测器的 AI 正样本空间"高度重叠，换风格难以逃出分布
- **溯源**：Private local case omitted from public package。
- **检测信号**：
  - 多轮改写后**分章节 AI 率反向移动**（此涨彼跌）——不是整体单调下降
  - 同份文本连续两轮命中的 AI 模式从一类迁到另一类（A28→A38 / A31→A36 等），但总命中数没减
  - 作者本人回读觉得"更顺了"——数字告诉相反故事
- **与 A37 的分工**：
  - A37 抓静态分布突变（hybrid seam，一篇里多种分布）
  - A39 抓动态分布迁移（改写过程中从一种 AI 分布滑到另一种）
  - 两者都反映同一底层问题：LLM 和检测器共享一个认知空间
- **应对策略**（元规则，不是段级改写）：
  - ①**每轮必测分章节 AI 率**，不只看整体字符数或总分
  - ②**对比前轮命中模式代码**：本轮 vs 前轮，若只是从 A 换到 B 但总命中数没减，是 A39 触发
  - ③**引入外部视角**：让**不同训练分布的 LLM** 做反审（Claude ↔ Codex / ↔ Gemini）——见 `references/human-writing-anchors.md` 或知识库 `跨LLM盲点互审`
  - ④**换锚点不换方向**：目标函数从"打破当前 AI 模式"换成"向真实学术作者分布对齐"（参照 `human-writing-anchors.md`，不是反向列举 AI 模式）
  - ⑤**复合扫**：A37 + A38 + A39 一起做——A37 查静态分布、A38 查新模板、A39 查动态迁移
- **不可抗例外**：
  - 第一轮改写可不做 A39 自检——原稿 AI 味集中时，打破一种模式仍有净收益
  - 2+ 轮起必须打开 A39 自检

## B. 学术回收规则（Academic Recovery）

### B1 作者位中性
- 不注入第一人称意见、叙事或情绪评价。

### B2 证据强度守恒
- 保留不确定性等级。
- 不把试探性结论改写成确定性结论。

### B3 术语与领域短语守恒
- 有效术语不得因"去 AI"而降精度。
- 例：`robust optimization` 保持原样。

### B4 可检验性优先
- 优先输出可验证、可复述、可追踪的陈述。

### B5 局部具体性优先
- 在源文本有支撑时，优先回写对象、条件、方法、边界与结果。
- 无支撑时删除空泛总括，不补虚构细节。

## 团队审稿视角映射

这不是三个独立审稿人，而是同一次审查的三个检查侧面。不要期望后一个侧面能发现前一个遗漏的问题——它们共享同样的盲区。

- `综合取舍层`：负责最终取舍，压缩多侧面意见，避免成稿残留多种风格拉扯。
- `结构侧`：重点检查 A7、A11、A12、A13、A14、A15、A18、A19、A20、A24、A28、A29、A33、A34、A37、**A38**。
- `证据侧`：重点检查 B2、B3、B5，并回看 Gate 0 的所有不变量；**A35（贴源改写的证据骨架）** 也归证据侧。
- `自然度侧`：重点检查 A1、A2、A4、A5、A6、A9、A10、A16、A17、A25、A27、A36、**A38**；参照 `references/human-writing-anchors.md` 判断。
- `分布迁移侧（2026-04-23 第四轮新增）`：重点检查 **A39（换风格陷阱）**——对比本轮 vs 前轮的命中模式代码与分章节 AI 率；仅在 ≥2 轮改写时启用。
- 若多侧面意见冲突，按 `证据侧 > 综合取舍层 > 其他侧面` 决策。
- 若未启用多代理，主代理必须按以上顺序完成串行自审。

## C. 模式选择矩阵

- `aggressive`：A1-A29 + A30-A39 + B1-B5（默认）。
- `balanced`：A1-A10、A13-A21、A24-A25、A29、A35、A36、A37、A38、A39（≥2轮起）+ B1-B5。
- `safe`：A5、A6、A8、A9、A10、A13 + B1-B5。

**说明**：A35（贴源改写）、A36（作者性抽薄）、A37（混合作者接缝）、A38（精炼反思型学术）在 balanced 也启用——A35/A36 涉及学术诚信与作者性边界（不是纯风格问题），A37 在人机协作场景极常见（学生 + 导师 + AI），A38 是 2024+ LLM 默认输出风格（不区分模式几乎必命中），都必须兜底。A34 只在 aggressive 启用（reasoning-family 模型场景特定）。A39 是元规律不是段级模式，仅在 ≥2 轮改写后启用——第一轮做打破一种模式仍有净收益。

## D. 执行优先级

> **顽固高分片段专项流程**：若某片段经≥2轮改写后AIGC仍≥0.85，先用A30/A31/A32/A33/A34做深层策略，再叠加A24/A20。表层模式对此类片段边际递减。

**前置步骤（2026-04-23 新增）**：

- **0a 分布判断**：改写前先判断是同分布（整篇 AI 或整篇人写）还是混分布（hybrid）。方法：对 5-10 个随机段落做"高影响句定位"（删掉最可疑一句/一段后 AI 感是否明显下降）。结果决定走 A33（同分布）还是 A37（混分布）路径。
- **0b 源头判断**：若怀疑是 reasoning-family 模型产出（推理深度不一致、有穷举-排除结构、局部过度解释），优先标记 A34 命中区域；若怀疑是基于某篇 source 的贴源改写（论证骨架紧跟某文献），优先标记 A35 命中区域。
- **0c 母骨架识别（2026-04-23 新增，顽固片段或章节级骨架嫌疑时启用）**：
  - **触发条件**：某章节 ≥3 轮改写后 AIGC 仍 ≥50%，或初诊发现章节级骨架嫌疑（四维度 parade、等量对称对偶、standalone takeaway 密集、CN/US 镜像对偶、H3 parade）
  - **操作**：不再逐个扫 A 模式。启动多 agent 辩论三叉戟（见 `multi-agent-debate-prompts.md`），让骨架审稿视角回答："如果只能打碎一个骨架就能让章节 AIGC 下降 10+%，你会打哪个？给出名称 + 为什么它是母骨架（其他 AI 信号都是它的表现）+ 打碎的最小动作"
  - **候选范围**：母骨架必定来自 A30（认知自足）/ A33（段间过滑）/ A35（贴源）/ A37（混合作者接缝）四个段际级模式
  - **章节类型对应**：Analysis/Conclusion 的母骨架大概率是 A30；Introduction 大概率是 A33；Literature Review 大概率是 A35(private local counterexample omitted; see section-specific strategy)
  - **识别后执行**：按该母骨架的专属最小动作（不是通用清单）设计手术：
    - 打 A30 → 删 standalone takeaway / H3 数减一 / 合并对偶段 / 段落外溢
    - 打 A33 → 删承接句 / 允许冷启动 / 主题轻微偏移
    - 打 A35 → 重组论证骨架按作者需要排序 / 一 source 多次引 / 证据交叉 / 术语簇替换
    - 打 A37 → 向全文基线重写突兀段 / 2-3 段锚点风格对齐
  - **元原则**：==打碎母骨架比打击所有下游模式更有效==。Private local case evidence omitted from public package.

**主流程**：

1. 先清除 chatbot 与填充痕迹（A5/A9）。
2. 再打散机械结构与平均化分布（A7/A11/A12/A13 + A24/A20/A25/A26）。
3. 中文本土对偶承接处理（A29）：中文稿件必做，英文稿件跳过。
4. 再压缩宣传化、空泛化与全景总括（A1/A2/A4/A14 + A28/A7）——**不删数字，只改叙述结构**；消融分析段叠加 A32；Gate 0 数字不可变。
5. 数值结果陈述重构（A27）——核心指标叙述，次要指标引用表格；打破"较X改善Y%"的重复句架。
6. 术语、证据强度与局部具体性回收（B1-B5）。
7. 深层结构与分布校准（A15-A19 + A20-A23）。
8. **作者性与贴源诊断（A35 + A36）**：这两条不等≥2轮顽固片段才做——它们涉及学术诚信与作者性边界，从第一轮就要扫。A35 重组论证骨架，A36 恢复作者立场。
9. 深层信号处理（A30/A31/A32/A33/A34）：适用于经≥2轮改写后 AIGC 仍≥0.85 的顽固片段。
   - **先 A30（段内）再 A33（段间）**：不同步做会过度改写。
   - **A34（推理模式接缝）**：仅在 reasoning-family 模型场景启用；命中时优先统一推理深度。
   - **A31 特别注意**：不仅避免教科书解释型，也要避免"比较选型框架"。
   - **A30 最强策略**：对于多轮顽固片段，采用场景置入法——彻底放弃原因链叙事，从具体失败观察切入。
10. **Hybrid 兜底（A37）**：如 0a 判定为混分布，在改写收尾前做风格基线一致化——选 2-3 段最像作者自己的段落作为锚点，让其他段落向锚点对齐。
11. **精炼反思型学术清理（A38）**：2024+ LLM 原生输出默认就是这种风格，每轮都要扫。按六类子模式（label+colon / editorial beat / editor voice / tutorial staircase / parade / aphoristic recap）各列清单，机械执行反精修策略——删接缝、反对称、标点节流、去 editor voice、留 unresolved edge、散文化收束。
12. **换风格陷阱自检（A39，第 2 轮起必做）**：改完后对比本轮命中模式代码 vs 前轮命中模式代码；若只是从 A 类换到 B 类但总命中数没减，说明触发了 A39。应对：换目标函数（从"打破 AI 模式"换到"向真实作者分布对齐"）+ 引入不同训练分布的外部 LLM 反审（见知识库 `跨LLM盲点互审`）。

**元原则 1：文意锚定优先于一切改写**

每次进入改写前，先对照该段的 Semantic Anchor（`references/semantic-anchor.md`）。核心命题、论证功能、信息重心是不可变约束——改写只能改变表达方式，不能改变段落在说什么、为什么说。多轮改写后尤其警惕累积漂移。

**元原则 2：结构优先，词汇次之**

实证规律：改写只换标记词/并列项措辞而保留等长完整并列结构时，AIGC 值几乎不下降。只有在结构层面做改动（合并项目、去掉粗体标题、制造详略差异、打破数字对仗）后，分值才出现实质变化。

> 每次进入改写前，先问：**"这段的文意锚是否保持？"** 再问 **"这段的并列结构是否对称？"** 最后才问 **"这段的措辞是否独特？"** 文意 > 结构 > 词汇。
