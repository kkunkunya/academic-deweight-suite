# 评分门控（V3 / 2026-04-23）

> **评估目标四层叠加**（2026-04-23 新增元判断）：
> 1. 语言层（Gate 1 维度 1-6）：表层/结构/分布级 AI 模式残留
> 2. 文意层（Gate 1 维度 7）：文意漂移
> 3. 改写痕迹层（Gate 1 维度 8）：humanizer 残片 + 高影响句集中度 + 分布逆变
> 4. 合规层（Gate 3）：披露完整性 + 过程证据可追溯 + 平台分数阈值
>
> 一个文本可以在 1-3 层过关，仍在第 4 层失败（未披露 → 期刊拒稿）。

## Gate 0：完整性硬门控（Hard Fail）

出现任一条即失败：
- 引用 token 或 citation key 被改动。
- 未经用户许可，数字、单位、日期、不等式、公式 token 被改动。
- 术语或实体名改动导致语义改变。
- 新增源文本未支持的事实性主张。
- 为了"更像人写"而补入源文本未提供的具体细节、人物、时间、引语或案例。
- **G0.6 引用精度核查失败**（2026-04-23 第三轮新增 / status: signal / 来源 Springer Nature 2025-04 自建检测工具，待核验）：引用存在但论断错配、页码错、年份错、版本错、作者名拼错、被引用论文根本不存在——"幻觉引用"问题。

若失败：回退问题片段，按更保守策略重写。

### G0.6 引用精度核查操作（前置门控，不等降重完成才查）

1. 随机抽 3-5 条核心引用（尤其支撑关键论断的那几条）。
2. 对照原文核查：论断方向是否与原文一致？页码/年份/版本是否匹配？作者名/标题是否正确？该论文是否真实存在？
3. 若任一不匹配 → Gate 0 硬失败，回退该引用所在片段，重新核源后再改写。
4. **降重前应确认 citation 本身是真的**——否则是拿假前提去打磨。
5. 判据引用：列出已核验的引用清单（被引论文标题 + 关键论断 + 原文对应页码）。

## Gate 1：AI 残留二元诊断

每个维度回答 yes/no，且必须引用文本中的具体句子作为判据。
"未见"不等于通过——必须有文本依据支持"未见"的判断。

### 维度 1：模板短语残留
- 诊断：文中是否仍存在空泛导语、万能总结或客服腔？
- 引用依据：若 yes，列出具体句子；若 no，明确说明检查依据。

### 维度 2：结构与节奏机械性
- 诊断：文中是否仍有重复句架、三连排比（含去掉显式标记词后仍对称的隐式三段平行）、均匀句长或刻意换词？是否存在操作-评价双句紧跟（A20）？列表条目是否格式完全同构、无详略差异（A21）？**是否存在公式-散文五段式重复（引出→公式→变量定义→物理意义→衔接）**？**是否存在≥4项散文化目录列举（连续多项等量描述模块/章节/组件，每项恰一句）**？
- 引用依据：若 yes，列出具体位置；若 no，明确说明检查依据。

### 维度 3：抽象空泛与范围膨胀
- 诊断：文中是否仍有营销词、无证据宏大表述或虚假范围？
- 引用依据：若 yes，列出具体句子；若 no，明确说明检查依据。

### 维度 4：深层 AI 感 + 覆盖感与篇幅分配（A15-A19 + A30 + A33 + A34；2026-04-23 吸收原维度 5a）
- **4a 深层结构**：是否仍有论证结构模板化、语气均匀性、连接词过密或抽象层次平推（A15-A19）？每个段落是否都是自足认知单元（A30）？段间衔接是否过于顺滑（A33）？
- **4b 面面俱到与篇幅分配**：各子话题是否被赋予近似等量篇幅？篇幅分配是否与话题重要性成比例？
- **4c 推理模式接缝**（reasoning-family 模型场景）：是否存在同一篇文本内推理深度不一致（段落间 polished 与 deliberate 交替）？（A34）
- 引用依据：若 yes，列出具体表现；若 no，明确说明检查依据。

### 维度 5：立场-介入-作者性坐标系（2026-04-23 重构，从 hedge 单维度扩展为 4 子项）

参照 `references/human-writing-anchors.md` 锚点 10-11。篇幅分配原 5a 已归位到维度 4b。

- **5.1 Stance variance**（立场强度分布 + 方向性偏差，2026-04-23 第三轮新增 5.1.b）：
  - **5.1.a 分布均匀性**：hedge（`may/可能/或许/大致`）与 booster（`must/明显地/必然/毫无疑问`）在段落间的密度变异。均匀或单一化 → yes
  - **5.1.b 方向性偏差**（第三轮新增 / status: signal / 来源待核验）：全文 booster 密度显著高于 hedge 密度，或出现"不符合科学常理的确定性"与"道德紧迫感"语气 → yes。示例信号：`X 是必然的 / 毋庸置疑 / 理应 / 必须 / 不可否认` 等高确信度词过密；在证据不充分的场景（讨论节、限制节）仍保持强断言。真实作者在证据不足处必然有犹豫。
  - 对照锚：真实作者在证据强处用 booster、不确定处用 hedge，且 hedge 词类型分散（不会整篇只用 `may`）
  - 判据引用：若 5.1.a yes，给出 3 个以上段落的 hedge/booster 清单对比；若 5.1.b yes，列出过度确信句的具体位置 + 证据是否充分

- **5.2 Engagement variance**（读者介入）：
  - 诊断：直接与读者互动的句式（`we can see/readers should note/值得注意的是/可以观察到`）密度分布。缺失或均匀过低 → yes
  - 对照锚：真实作者在关键论断处呼唤读者，一般陈述处不呼唤——形成分布差异
  - 判据引用：列出直接互动句的位置分布；若缺失，说明"全篇 polished passive voice"依据

- **5.3 Counter-position pressure**（对立观点处理）：
  - 诊断：对反对意见、竞争方案、反例的处理力度。全文缺少反驳段落或对立观点被一笔带过 → yes
  - 对照锚：真实作者会正面回应竞争方案甚至展开反驳段；AI 文本倾向"平衡陈述"，缺锋利度
  - 判据引用：列出文中所有涉及"对立观点"的位置 + 处理方式

- **5.4 Author ownership density**（作者立场密度，对应 A36）：
  - 诊断：作者作为论断主体的直接立场表达密度（`本文采用 X 的理由是...` / `我们认为...` / `本文不采用 Z 因为...`）。密度过低或全篇外包给"研究表明/数据显示" → yes
  - 对照锚：真实作者在方法选型、参数选择、结果解读的关键节点有直接立场表达
  - 判据引用：列出方法节/讨论节的作者直接立场表达位置 + 密度；若密度过低，给出应补充的 2-3 处

### 维度 6：信息密度均匀性
- 诊断：是否存在段落≤400字符但包含≥3个独立事实/数字/百分比，且每句信息密度近似相等（无过渡句、无主次、每句等权填塞）？是否存在全文数值结果陈述均以"指标名称+数值，较baseline改善X%"相同格式重复？
- 引用依据：若 yes，列出具体段落并说明密度问题；若 no，明确说明检查依据。

### 维度 7：文意漂移
- 诊断：改写后的段落核心命题是否与 Semantic Anchor 一致？论证功能是否保留？信息重心是否偏移？段间论证承接是否断裂？
- 方法：逐段对照 Semantic Anchor 的 5 个字段（核心命题、论证功能、信息重心、读者收获、上下文依赖）。
- 引用依据：若有漂移，列出原锚内容 vs 当前文本的具体偏差；若无漂移，说明对照依据。
- 特别注意：多轮改写后此维度权重上升——每多一轮，漂移累积风险越高。

### 维度 8：改写痕迹与高影响句集中度（2026-04-23 升级，status: signal）

**8a 多轮改写痕迹自检**（仅≥2 轮改写后启用）：
多轮改写本身会引入新的同构。溯源：Turnitin 2025-08 反 humanizer 更新 + GPTZero 3.7b。
- 检测信号：
  - 过渡词新同构：`然而/不过/值得一提的是/更进一步看/某种意义上` 等密度超过原文
  - 作者判断语过量：A30 倡导的"这一设置并不直觉，但……"等在每段都有，反成新模板
  - 毛刺感刻意化：A33 倡导的"段间毛刺"呈现为固定节奏（每隔 3 段必出现一次主题偏移）
  - hedge/booster 过度注入：为应对维度 5.1，密度抬升但变异度未提高
- 判据：若 yes，列出具体过渡词/判断语的出现频次 + 段落分布。

**8b 高影响句集中度**（对应 A37；从第一轮起启用）：
溯源：GPTZero Advanced Scan sentence impact + Turnitin/Originality 反 humanizer 观察——被洗过的文本典型是"整体中庸、局部露馅"。
- 诊断操作：
  1. 取 5-10 个随机段落
  2. 对每段问"如果删掉最可疑的一句，这段的 AI 感会明显下降吗？"
  3. 多数答案"会"→ 高影响句集中，AI 感集中在少数高风险区（hybrid 或 humanizer 残片）→ **改写应优先处理高影响句，而不是逐段平铺改**
  4. 多数答案"不会"→ 均匀分布（可能真人，也可能未洗原生 AI，结合其他维度判断）
- 判据：标出全文 5-10 个高影响句清单（位置 + 为何可疑）。

**8c 分布逆变痕迹**（从第一轮起启用）：
对应底层原则"分布逆变警觉"。
- 诊断：某段读起来"过分通顺"——句式圆滑到像从另一种表达分布翻回来的。
- 判据：若 yes，列出 2-3 个"过分通顺"的段落，说明通顺感来自哪里（句式统一、连接词完备、用词精准到无漏洞）。
- 改写策略：适度引入颗粒度差异——某处保留略粗糙但可读的表达，不追求每段都打磨到位。

**8d 长文本后期稳定性**（2026-04-23 第三轮新增 / status: signal / 来源待核验，arXiv 2601.04833 Late-Stage Volatility Decay）：
- 适用场景：长文本（>5000 字，学位论文/综述/长篇报告）才启用；短文本跳过。
- 核心信号：自回归生成的 AI 文本在序列后半段呈现稳定性——>50% 位置后的对数概率波动降低 24-32%。
- 诊断操作：
  1. 全文切为前 30%、中 40%、后 30% 三段
  2. 对每段估算：句长方差、词汇丰富度、句式结构多样性
  3. 若后段显著比前段稳定/平整（句长方差下降、词汇丰富度下降、句式趋于单一）→ yes
- 改写策略：不是让前段变稳，而是**让后段恢复波动**——后半段主动引入低频词、结构突变、意外转折。结论节/展望节最容易塌缩，警觉这两节是否比中段更单调。
- 判据引用：agent 不必精确计算方差，可用"前段句长在 8-35 字之间波动、后段集中在 20-28 字"这种语言估算描述。

**维度 8 整体适用场景**：
- 8a：仅 ≥2 轮改写后启用
- 8b、8c：从第一轮起启用
- 8d：仅长文本（>5000 字）启用

**维度 8 改写策略**：发现后不是"再加人感"，而是**减少刻意人感痕迹**——删掉多余的作者判断语、让部分段落回归平实陈述、优先处理高影响句而非平铺改写、允许部分段落保留表达颗粒度、让后半段恢复真实波动。

## 通过条件

- 维度 7（文意漂移）单独判定：若诊断为 yes，无论其他维度结果如何，必须先回退漂移段落再继续。
- 维度 1-6 + 维度 8：至多 1 个维度诊断为 yes → 通过；2 个或以上 → 仅重写标记片段，最多 2 轮。
- 维度 8 仅在 ≥2 轮改写后启用（第一轮不判定）。
## 显性残留自审（对比操作）

不要做”整体感觉有没有 AI 感”的扫描式自问——同一套认知评估同一输出，盲区重叠。

改为对比操作：
1. 随机取 3 个不相邻段落。
2. 对每段问：「如果这是真实论文的段落，审稿人会觉得哪里不对？」
3. 检查全文详略分布是否与内容重要性成比例（参照 `references/human-writing-anchors.md` 锚点 2 和 6）。

输出格式：
- 只输出具体问题位置（段落标识 + 问题描述）。
- 不输出整体评价或赞美性总结（”整体流畅”、”大幅改善”之类）。

未达通过条件：仅重写标记片段，最多 2 轮。

## Gate 2：团队共识门（启用 team_review 时）

- `证据侧` 不得报告完整性硬失败。
- `结构侧` 与 `自然度侧` 至少一方确认主要 AI 痕迹已明显下降。
- 若多侧面意见冲突，先回退到保真更强的版本，再局部修正文风问题。

## Gate 3：合规与过程追溯门（2026-04-23 新增，status: signal）

> 本 Gate 是**非语言学维度的 checklist**，不做文本诊断，只审查流程完整性。适用场景：投稿学术期刊、提交学位论文、公司受控文档。一个文本可在 Gate 0-2 全部过关但仍在此层失败（例如未披露 → 期刊拒稿）。

### G3.1 披露完整性 checklist

对照 IEEE / ACM / Elsevier / Springer Nature / Science 主流出版社 2025-2026 政策：

- [ ] 是否明确声明了 AI 使用？
- [ ] 披露内容是否包含：工具名 + 版本 + 使用环节？
- [ ] **披露是否区分 AI-Native 与 AI-Polish？**（2026-04-23 第三轮新增 / status: signal）
  - AI-Native（纯 AI 生成）：AI 独立起草，人类只做校对——合规风险最高
  - AI-Polish（人类起草 + AI 润色）：人类保留核心观点和逻辑框架，AI 只做语言层面润色——部分期刊视为合规
  - 披露应明确区分 `AI-assisted polish of human-drafted text` vs `AI-generated content`
  - 降重策略上应尽量保留原始人类核心观点，使特征贴近合规的 AI-Polish
- [ ] AI 是否未被列为作者？
- [ ] 若涉及 AI 生成图像：是否符合目标期刊政策？（Springer Nature 禁止未实质修改的 AI 图像；Science 禁止 AI-generated multimedia 除非编辑许可）
- [ ] **引用是否已完成精度核验？**（对应 Gate 0 G0.6）——幻觉引用在 Springer Nature 自建检测器下会被标记

### G3.2 过程证据可追溯性 checklist（中文期刊增强要求）

对照《时空信息学报》《农业经济问题》等中文期刊 2025 年公告：

- [ ] 原始 prompt 是否留档？
- [ ] 工具名称、开发者、版本、使用时间是否记录？
- [ ] 页面截图或使用记录是否可回溯？
- [ ] 若评审/老师追问"这段是怎么来的"，能否给出解释？

### G3.3 平台分数阈值（校级红线，依目标而定）

- 国内硕士论文：AIGC 率常见阈值 20-30%（依本校通知）
- 国内本科论文：AIGC 率常见阈值 30-40%
- 学校使用的检测平台（知网 AIGC / 维普 AIGC / PaperPass / 格子达）口径不一致，以本校指定平台为准
- 国际期刊：以目标期刊 AIGC 披露政策为准；部分期刊（如 Springer Nature）使用自研 detector，不以公共 detector 分数为主要依据

### Gate 3 通过条件

- 三段 checklist 全部勾选通过 → Gate 3 通过
- 任一段未通过 → 不是文本问题，而是**流程问题**——需要补披露声明、补留档材料、或调整到符合阈值
- Gate 3 不覆盖 Gate 0-2：语言层过关 + 合规层过关是必要条件，缺一不可

## 返工优先级

1. 完整性问题（先修复 Gate 0）。
2. chatbot/填充痕迹（模板感最高）。
3. 结构同构、节奏机械与刻意换词。
4. 宣传化、空泛化与范围膨胀。
5. 局部具体性与轻量精炼收尾。
6. 合规层（Gate 3）—— 语言层过关后补流程文件。
